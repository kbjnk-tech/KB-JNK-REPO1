## 1. 프로젝트 개요

은행 외환 창구 직원이 **환전(현찰 살 때/팔 때)·해외송금 비용 계산**과 **거래 기록 관리**를 한 화면에서 수행할 수 있는 웹 대시보드입니다.

| 구분 | 내용 |
|------|------|
| 목적 | 스프레드·우대율·JPY 단위 등 암산이 어려운 환율 계산을 자동화하고, 기록을 검색·삭제·내보내기 |
| 스택 | React 19 · Vite · TypeScript · Tailwind CSS v4 · Vitest · Supabase · Vercel |
| 아키텍처 | Feature-Sliced Design (FSD) |
| 기본 저장 | **localStorage** (새로고침 유지, 명세서 필수) |
| 클라우드 | (선택) Supabase — 설정 시 시작 로드·추가/삭제 자동 반영 |
| 배포 | Vercel 공개 URL |

정본 요구·기술 문서는 루트 `prd.md` / `trd.md` 및 `docs/`에 정리되어 있습니다.

## 2. 설치 및 실행 방법

### 2.1 사전 준비

- Node.js **20+** / npm
- (선택) 한국수출입은행 OpenAPI 키, Supabase 프로젝트

### 2.2 설치

```bash
# 1) ZIP 압축 해제 후 프로젝트 폴더로 이동
cd KB_전나경_외환도우미   # 또는 압축 해제된 폴더명

# 2) 의존성 설치 (node_modules는 ZIP에 포함하지 않음)
npm install

# 3) 환경변수 파일 생성
cp .env.example .env
```

`.env`에 아래 키를 채웁니다. (실키는 커밋·제출 ZIP에 넣지 마세요.)

| 키 | 용도 | 필수 |
|----|------|------|
| `VITE_EXCHANGE_API_KEY` | 한국수출입은행 현재환율 OpenAPI | 실시간 환율 버튼용 (없으면 수동 입력) |
| `VITE_SUPABASE_URL` | Supabase 프로젝트 URL | (선택) 클라우드 동기화 |
| `VITE_SUPABASE_ANON_KEY` | Supabase anon/publishable key | (선택) 클라우드 동기화 |

### 2.3 로컬 실행

```bash
npm run dev
```

브라우저에서 표시되는 주소(보통 `http://localhost:5173`)로 접속합니다.

### 2.4 테스트 · 빌드

```bash
npm test          # Vitest — PRD 인수 시나리오 회귀 (12케이스)
npm run build     # 타입체크 + 프로덕션 빌드
npm run preview   # 빌드 결과 미리보기
```

### 2.5 Supabase 테이블 (선택)

Supabase Dashboard → SQL Editor에서 `supabase/migrations/20260716_create_transactions.sql` 내용을 실행합니다.  
env가 설정되어 있으면 앱 시작 시 클라우드에서 기록을 불러오고, 추가/삭제 시 자동 반영됩니다.  
미설정·연결 실패여도 **localStorage**만으로 정상 사용합니다 (명세서 필수 요건).

### 2.6 배포본 확인

별도 설치 없이 https://kb-fx-teller-helper.vercel.app 에서 핵심 기능을 확인할 수 있습니다.

## 4. 선택한 아키텍처 (FSD)와 이유 · 레이어/의존 규칙

### 4.1 선택: Feature-Sliced Design (FSD)

**이유**

1. 화면·기능 단위로 폴더를 나누어 **AI Rules로 수정 범위를 강제**하기 쉽습니다.
2. 환율·송금 **계산을 순수 함수(`entities`/`shared`)로 분리**해 Vitest·CodeRabbit 검증에 유리합니다.
3. 송금·CSV·API·Supabase 등 가산 기능을 **같은 레이어 규칙**으로 확장할 수 있습니다.

### 4.2 레이어 구조

```text
src/
├── app/          # 앱 엔트리 조합
├── pages/        # 대시보드 페이지 조합
├── widgets/      # exchange / remittance / transaction-history 패널
├── features/     # calculate-*, add/search-transaction, export-csv, sync-supabase
├── entities/     # currency, rate, transaction (도메인)
└── shared/       # lib, ui, config, api, supabase
```

### 4.3 의존 규칙

1. **상위 → 하위만** import: `app → pages → widgets → features → entities → shared`
2. **동일 레이어 슬라이스 간 import 금지** (예: feature ↔ feature 직접 참조 금지)
3. 외부 노출은 슬라이스 **`index.ts`(Public API)** 만 — deep import 금지
4. 환율·송금·수수료 계산은 UI·localStorage·API와 섞지 않은 **순수 함수**

## 5. 표준 Cursor Rules 세트와 표준 Workflow

### 5.1 Rules (`.cursor/rules/`)

| 파일 | 적용 | 역할 |
|------|------|------|
| `00-architecture.mdc` | Always | FSD 의존·Public API·작업 범위 |
| `01-collaboration.mdc` | Always | Plan/Implement/Review·시크릿·MCP·PR |
| `10-domain-fx.mdc` | entities, shared/lib·config | 도메인 공식·반올림·수수료 |
| `20-ui.mdc` | pages, widgets, shared/ui | UI 셸 (features 제외) |
| `25-client-infra.mdc` | api, supabase, sync | FE↔인프라 통신 경계 |
| `30-testing.mdc` | `*.test.ts(x)` | TDD·회귀 |
| `40`~`43-feature-*.mdc` | 해당 feature/widget | 슬라이스별 허용/금지 경로 |

원칙: Always = 협업·공통 가드레일. File-specific = 해당 파일을 다룰 때만 중첩 적용.

### 5.2 Workflow (`docs/WORKFLOW.md` · `docs/prompts.md`)

```text
Plan → Implement → Review
```

| Mode | 목적 |
|------|------|
| **Plan** | 범위·파일·리스크만 정리, 대량 코드 변경 금지 |
| **Implement** | 허용 경로만 수정, FSD·feature 룰 준수, context7로 최신 API 확인 |
| **Review** | FSD·PRD 인수·테스트·범위 일탈 점검 |

기능 단위로 브랜치·PR을 나누고 CodeRabbit 자동 리뷰를 받는 흐름을 사용했습니다. (예: PR #1)

## 6. 구현한 고급 기능

### 6.1 제품·인프라 가산 기능

| 항목 | 구현 요약 |
|------|-----------|
| 해외송금 탭 | `widgets/remittance-panel` + `entities/rate` 총 출금액 |
| 실시간 환율 API | `shared/api/fetchBaseRate`, Vite/Vercel `/api/exim` proxy·rewrite, 실패 시 수동 입력 |
| CSV / 인쇄 | UTF-8 BOM CSV, `@media print`로 거래 기록만 출력 |
| 다크모드 / 반응형 | `ThemeToggle` + Tailwind `dark` / `sm`·`lg` |
| Vitest (TDD) | PRD ①~⑤·우대 100%·송금 4,022,600원 등 12케이스 |
| CodeRabbit | `.coderabbit.yaml` + GitHub PR 자동 리뷰 (PR #1) |
| MCP | `.cursor/mcp.json` — filesystem, context7 |
| Custom Mode | Plan / Implement / Review + WORKFLOW·prompts |
| Supabase | `transactions` 자동 동기화(선택) + localStorage 필수 (E-7.1) |
| Vercel | https://kb-fx-teller-helper.vercel.app |

### 6.2 AI 생산성·협업 도구

| 도구 | 활용 |
|------|------|
| **Cursor** | 구현의 중심 IDE. Rules·MCP·Agent로 FSD 범위 안에서 기능 단위 구현·리뷰 |
| **NotebookLM** | 프로젝트 명세(PRD/TRD 등)를 분석하고, 요구사항 및 개발 우선순위를 정리하는 데 활용. 구현 전에 기능 목록과 개발 계획을 수립하는 데 도움을 받음 |
| **로컬 LLM** | Ollama 등 로컬 LLM도 비교·검토해 보았으나, **실제 프로젝트 구현은 Cursor를 중심으로 진행** |
| **CodeRabbit** | 기능 단위 PR에 대한 자동 코드 리뷰로 문서·체크리스트 정합성 보완 |
| **context7 MCP** | React/Vite/Supabase 등 최신 API 문서 조회 |
