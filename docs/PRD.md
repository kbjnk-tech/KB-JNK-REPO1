# Product Requirement Document (PRD)

> 제출 시 위치: `docs/PRD.md`  
> 프로젝트: 외환 창구 업무 도우미 (FX Teller Helper)  
> 폴더: `KB_전나경_외환도우미`

---

## 1. 프로젝트 개요 (Overview)

### 1.1 배경

은행 외환 창구 직원은 매일 **환전 금액 계산**(현찰 살 때/팔 때, 우대율 적용)과 **거래 내용 기록·조회**를 반복한다. 환율·스프레드·우대율·통화별 단위(예: JPY 100엔당 고시)가 얽혀 암산이 어렵고, 거래 기록을 따로 정리하는 일도 번거롭다.

### 1.2 제품 목적

환전·해외송금 비용 계산을 자동화하고, 거래 기록을 한 화면에서 추가·검색·삭제·내보내기할 수 있는 **단일 대시보드형 외환 창구 도우미 웹앱**을 제공한다.

### 1.3 학습·조직 목적 (과제 핵심)

- FSD 폴더 구조로 Context 범위를 제한하고, 레이어 간 의존 방향을 규칙으로 강제
- 팀 표준 Cursor Rules·Workflow 문서로 일하는 방식을 표준화
- AI 위임·협업(CodeRabbit, MCP, Custom Mode) 과정을 문서화
- 도메인 순수 함수 + Vitest(TDD)로 계산 정확도를 회귀 검증

### 1.4 범위 원칙

본 문서는 **필수(MVP) + 가산(고급) 기능을 모두 구현 범위에 포함**한다.  
디자인 완성도는 평가 대상이 아니며, **기능 구현 + 아키텍처 준수**가 평가 대상이다.

## 2. 사용자 및 시나리오

| 항목 | 내용 |
|------|------|
| 주요 사용자 | 은행 지점 외환 창구 담당 직원 |
| 사용 환경 | 데스크톱 브라우저 (반응형이면 가산 품질↑) |

**핵심 시나리오**

1. 고객이 외화 환전(현찰 매입/매도)을 요청하면 직원이 환전 계산기를 사용한다.
2. 매매기준율·스프레드율·우대율·금액을 입력해 적용환율과 원화 금액을 확인한다. (실시간 환율 API로 기준율 자동 채움 가능)
3. 해외송금이면 전신환 적용환율 + 송금수수료 + 전신료로 **총 출금액**을 확인한다.
4. 확정 거래를 기록에 등록하고, 키워드로 검색·삭제하며, 필요 시 CSV로 내보낸다.
5. 새로고침 후에도 기록이 유지되며(localStorage), 클라우드(Supabase)에도 동기화할 수 있다.

## 5. 예외·경계값 (Error Handling)

| ID | 입력/상황 | 기대 동작 |
|----|-----------|-----------|
| E-1.1 | 매매기준율·금액이 빈 값·0·음수·문자 | 계산하지 않고 안내 메시지 |
| E-1.2 | 우대율 0% | 스프레드 전액 적용 (정상) |
| E-1.3 | 우대율 100% | 적용환율 = 매매기준율 |
| E-1.4 | 우대율 음수 / 100 초과 | 0~100 제한 또는 안내 |
| E-1.5 | 숫자 필드에 문자·특수문자 | 숫자만 허용 또는 무시 |
| E-2.1 | 빈 내용으로 거래 기록 추가 | 추가 거부 + 안내 |
| E-5.1 | 환율 API 실패 | 수동 입력 유지 + 오류 안내 |
| E-7.1 | Supabase 연결 실패 | localStorage로 계속 사용 가능 + 안내 |

## 6. 인수 기준 (Acceptance Criteria)

기준 스프레드: **현찰 1.75%**, **전신환 1.0%**  
반올림 순서: **적용환율 소수 2자리 반올림 → 곱셈 → 원화 정수 반올림**

### 6.1 환전·전신환 계산

| # | 통화 | 거래구분 | 매매기준율 | 우대율 | 금액 | 적용환율 | 원화 금액 |
|---|------|----------|------------|--------|------|----------|-----------|
| ① | USD | 현찰 살 때 | 1,540.00 | 0% | $500 | 1,566.95 | 783,475원 |
| ② | USD | 현찰 살 때 | 1,540.00 | 80% | $500 | 1,545.39 | 772,695원 |
| ③ | USD | 현찰 팔 때 | 1,540.00 | 0% | $500 | 1,513.05 | 756,525원 |
| ④ | JPY | 현찰 살 때 | 980.00 (100엔당) | 0% | ¥100,000 | 997.15 | 997,150원 |
| ⑤ | USD | 송금(전신환) | 1,540.00 | 0% | $1,000 | 1,555.40 | 1,555,400원 |
| 추가 | USD | 현찰 살 때 | 1,540.00 | 100% | $500 | 1,540.00 | **770,000원** |

> 우대 100%이면 스프레드 0 → 적용환율 = 매매기준율. `$500 × 1,540 = 770,000원`.

### 6.2 해외송금 총 출금액

**시나리오:** $3,000 / 매매기준율 1,320 / 우대 0% / 전신환 스프레드 1.0%

| 항목 | 값 |
|------|-----|
| 적용환율 | 1,333.20 |
| 송금원금(원화) | 3,999,600원 |
| 송금수수료 | 15,000원 |
| 전신료 | 8,000원 |
| **총 출금액** | **4,022,600원** |

### 6.3 거래 기록

| 동작 | 예상 결과 |
|------|-----------|
| 기록 2~3개 추가 | 목록에 즉시 표시 |

| 키워드 검색 | 해당 기록만 필터 |

| 빈 내용 추가 | 거부 |
